## ✅ 今天完成的主要工作

### 1. 配置系统架构 ✅
**目标**: 实现以仓库根目录 `config.yaml` 为唯一配置源的安全配置系统

#### 核心架构设计:
- **唯一配置源**: 仓库根目录 `/config.yaml` 作为所有配置的源头
- **安全去敏**: 前端只能访问去敏后的配置（移除敏感字段如 `database`）
- **运行时加载**: 前端运行时通过 `fetch('/config.yaml')` 动态加载配置
- **同步机制**: 构建时自动同步配置到 `frontend/public/config.yaml`

#### 配置文件结构:
```yaml
# 仓库根目录: /config.yaml
api:
  baseUrl: http://localhost:3001/api

assets:
  strategy: signedUrl
  publicBaseUrl: https://via.placeholder.com
  sign:
    mode: mock
    expiresSeconds: 300

community:
  url: https://hub.captraw.com

# 后端专用配置（不会同步到前端）
database:
  url: postgresql://...
objectStorage:
  endpoint: ...
  bucket: ...
```
